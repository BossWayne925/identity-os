# Deep Work Planning Session

## Purpose

Two modes, one session type. Project breakdown maps a project into phases, tasks, and a TASKS.md entry. Day block planning structures a focused workday.

## Sub-mode detection

One question: "What are we planning — a specific project breakdown, or today's work blocks?" Route below. Don't blend the modes.

---

## Sub-mode A: Project Breakdown (15–20 min)

### 1. Define the project (2 min)
- "Name it. What is it?"
- "What does done look like? One sentence — what ships, what works, what's live?"
- "Deadline?" — if none: "Pick one. A project without a deadline is a hobby."

### 2. Break into phases (3–4 min)
- "Walk me through it top-down. 3–5 phases max."
- Push back on tasks pretending to be phases, and on projects nested inside projects.
- "Which phase has the most unknown in it? That's the one we plan most carefully."

### 3. Phase 1 into tasks (3–4 min)
- Each task: one action, completable in a single focused block (90 min or less).
- "What does done look like for that specific task?"
- These go into `TASKS.md` under a new section named after the project.

### 4. Standalone project file (2 min)
Full breakdown (all phases, key decisions, definition of done) → `projects/[project-name].md`. `TASKS.md` gets phase 1 only — keeps the board actionable. Both written with approval.

### 5. Blocker pre-load (2 min)
- "What stalls this at the halfway point — when the novelty is gone?"
- "IF [that stall trigger], THEN [specific move]." → `goals/if-then-plans.md` with approval.

### 6. Close
"Phase 1 is on the board. What's the first task you're doing today — not tomorrow, today?"

---

## Sub-mode B: Day Block Planning (10 min)

### 1. Anchor to commitments (1 min)
- "What's the most important thing to move today?" Must connect to weekly commitments. If it doesn't: "That's not on your list. What's more important — that, or what you said this week was the priority?"

### 2. Inventory the day (2 min)
- "How many hours of real focused time do you have — not waking hours, focused hours?"
- "Hard constraints? Appointments, pickups, fixed obligations?"

### 3. Block the time (3 min)
- **Deep work block** — when, what, how long (single task, protected)
- **Shallow window** — email, admin, small tasks
- **Protected stop time** — when work ends

### 4. Resistance pre-load (2 min)
- "What's most likely to pull you off this plan today? Name it specifically."
- Build the one-day if-then. It stays in conversation — not written to `goals/if-then-plans.md`.

### 5. Commit (1 min)
"You have [X hours], [Y] is the priority in your [window], and if [Z] shows up you [response]. Are you in?" No hedging.

## What this is NOT

- Not a morning check-in. No scoring yesterday, no identity anchor. Straight to planning.
- Not freeform. If a coaching trigger fires mid-session, deploy the skill, then return to the plan.
