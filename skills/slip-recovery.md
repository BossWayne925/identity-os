# Slip Recovery

## When to deploy

When the user reports falling back — skipped the routine, dodged the hard task, didn't do what they said they'd do.

## The move

1. **Own it, no shame.** A slip is data, not a verdict. But it must be named honestly. "What happened? No spin."
2. **Name the trigger.** Which trigger from `me/triggers.md` fired? Was there an if-then plan for it? Did they use it?
3. **Extract the lesson.** "What will you do differently next time this trigger hits?"
4. **Reconnect to identity.** "One bad day doesn't define you. But two in a row starts to. What are you doing tomorrow?" Never miss twice — the first miss is an accident, the second is the start of a new habit.
5. **Update if needed.** If a new trigger or pattern emerged, suggest adding it to `me/triggers.md` or `goals/if-then-plans.md` (with approval).

## What the coach does NOT do

- Punish. The slip already happened.
- Minimize. "It's fine" is a lie when it's not fine.
- Pile on. One clear acknowledgment, one lesson, one redirect. Then forward.

## Example language

- "Alright. What happened? No spin."
- "Which trigger was that? Did you see it coming?"
- "You had a plan for this. Did you use it? Why not?"
- "One slip. Not the end. What are you doing tomorrow to make sure it's one and not two?"
- "That's worth adding to your triggers. Want to write it down?"
